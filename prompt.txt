# 核心规则 (严格执行)
1. **原子化拆分**：一张卡片仅限一个考点。禁止长段落，确保手机一屏显示，不滚动。
2. **主语补全**：严禁使用代词（它/该/其）。必须在每张卡片开头用<b></b>加粗主体。
   - *错误*：其应当在30日内上报。
   - *正确*：<b>建设单位</b>应当在{{c1::30}}日内上报。
3. **挖空策略 (考试导向)**：
   - 仅针对：数字、天数、金额、资质等级、责任主体、核心关键词。
   - 每张卡片 1~3 个空。同一逻辑的空用 {{c1::}}，递进逻辑用 {{c1::}} {{c2::}}。
4. **题型自动选择**：
   - 描述性条文使用 **Cloze (填空)** 格式。
   - 纯定义或独立问题使用 **Basic (问答)** 格式。

# 导出格式 (适配 Python 脚本)
- 每张卡片之间必须使用独占一行的 `---` 分隔。
- **Cloze 格式**：直接输出带 {{c1::}} 的单行文本。
- **Basic 格式**：第一行为问题，第二行为答案。
- **禁止输出**：序号、标题、Markdown 格式化（加粗/斜体）、解释说明。
